## 전화번호 인증 (필수 구현)

**사용자 앱 로그인 = 전화번호 + SMS OTP**

| 단계 | API                       | 설명                                                                          |
| ---- | ------------------------- | ----------------------------------------------------------------------------- |
| 1    | `POST /auth/phone/send`   | Body: `{ "phone": "01012345678" }` → SMS 인증번호 발송                        |
| 2    | `POST /auth/phone/verify` | Body: `{ "phone", "code": "123456" }` → 검증 후 accessToken/refreshToken 반환 |

- OTP 6자리
- 기존 사용자: 로그인 / 신규: 자동 가입 후 로그인
- 가입 시 10,000원 마일리지 지급

## 대리호출 지도·출발지 (BE 참고)

앱 **대리호출 지도**에서 출발지는 다음 경로로 결정되며, 모두 **`POST /rides/call`의 `latitude` / `longitude`에 그대로 반영**됩니다.

1. **출발지 검색** (`/geocode/search`, `/geocode/keyword` 등 + 카카오 로컬 보조): 사용자가 고른 POI의 위·경도
2. **GPS 새로고침**: 현재 위치
3. **위치 상세수정**: 지도 중심 좌표로 확정 후, 확인 시 **`GET /geocode/reverse?lat=&lng=`** 로 주소 보강 (실패 시 문구 `"지도에서 선택"`)

도착지를 아직 고르지 않아도 출발만 검색·상세수정·호출 가능합니다.

### `POST /rides/call` 바디 — 앱이 보내는 값 (필드 의미)

| 필드                                               | 앱에서의 의미                                                                                                              |
| -------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------- |
| `latitude`, `longitude`                            | **출발(픽업) 좌표** (WGS84, double)                                                                                        |
| `address`                                          | **출발지 표시용 문자열** (검색 결과명, 역지오코딩 합본 주소, 또는 `"지도에서 선택"`)                                       |
| `addressDetail`                                    | **도착지 관련 문자열**: 도착에 `address`가 있으면 그 값, 없으면 도착 **장소명(`name`)**. **도착 미선택 시 빈 문자열 `""`** |
| `estimatedDistanceKm`, `estimatedFare`, `fareType` | 도착이 있을 때만 전송, 없으면 **null(미포함)**                                                                             |
| `paymentMethod`, 옵션 필드, `phone` 등             | 기존과 동일                                                                                                                |

> 참고: 스펙 예시에 `addressDetail`이 “상세주소”로만 적혀 있어도, **현재 앱은 도착 설명에 가깝게 채웁니다.** 배차·CS 화면에서 출발/도착을 나눌 때 `latitude`/`longitude`+`address`는 출발, `addressDetail`+별도 도착 좌표가 필요하면 앱 스키마 확장 검토가 필요합니다. (현재 앱은 도착 좌표를 이 API 바디에 **별도 필드로 보내지 않음** — 필요 시 FE에서 `destinationLatitude` 등 추가 가능.)

### 역지오코딩 `GET /geocode/reverse`

- 앱은 응답에서 `data.address`, `data.addressDetail`, `data.region` 형태를 파싱합니다. **빈 응답 시** 카카오 REST 키가 있으면 클라이언트에서 카카오 역지오만 시도합니다.
- 상세수정 직후 호출 횟수가 늘 수 있으니 **레이트 리밋·캐시** 있으면 유리합니다.
